# 物模型设计指南
设计物模型（Thing Model）是物联网（IoT）系统中**标准化、结构化描述设备能力和行为**的核心环节，本质是为物理设备建立一套“数字孪生”的抽象模型，让平台和应用能统一理解、管理、交互不同类型的设备。

## 一、物模型设计的核心原则
1. **标准化**：字段命名、数据类型、单位等遵循行业规范（如阿里云IoT、华为IoT、MQTT-SN等标准），避免“一物一规”；
2. **最小必要**：只定义设备实际具备的能力，不冗余（比如温湿度传感器不定义“转速”属性）；
3. **可扩展**：预留扩展字段/版本号，支持设备固件升级后新增能力；
4. **易理解**：命名直观（如`temperature`而非`t1`），附带描述，降低开发和维护成本；
5. **分层抽象**：按“属性、服务、事件”分层，区分“静态特征、主动操作、被动通知”。

## 二、物模型的核心组成（通用标准）
无论基于哪个IoT平台（阿里云、华为云、腾讯云），物模型都包含三大核心模块，这是行业通用范式：

| 模块       | 定义                                                                 | 示例                          |
|------------|----------------------------------------------------------------------|-------------------------------|
| **属性（Property）** | 设备的静态/动态状态，可读写（如设备型号、实时温度），是“是什么”       | 设备型号（只读）、当前湿度（可读可写/只读） |
| **服务（Service）** | 设备可执行的操作，需主动调用（如重启、校准、设置阈值），是“能做什么” | 重启设备、设置温度报警阈值    |
| **事件（Event）**   | 设备主动上报的异常/状态通知（如报警、故障），是“发生了什么”           | 温度超限报警、低电量提醒      |

### 各模块的细化属性
- **属性**：需定义标识符（唯一）、数据类型（int/float/bool/string/枚举）、单位（℃/%）、读写权限（R/RW）、取值范围（如温度0-100℃）；
- **服务**：需定义入参（如设置阈值的“目标值”）、出参（如操作结果“成功/失败”）、调用方式（同步/异步）；
- **事件**：需定义级别（信息/警告/错误）、携带数据（如超限报警的“当前温度”）、上报触发条件。

## 三、物模型设计的具体步骤（以温湿度传感器为例）
### 步骤1：梳理设备能力（需求调研）
先明确设备的核心功能，避免遗漏或冗余：
- 静态特征：设备型号、固件版本、设备SN码（只读）；
- 动态状态：实时温度（只读）、实时湿度（只读）、采样频率（可读可写）；
- 可执行操作：重启设备、校准温湿度、设置采样频率；
- 异常通知：温度超限（警告）、湿度超限（警告）、设备离线（错误）、低电量（警告）。

### 步骤2：结构化定义各模块（标准化）
按“唯一标识符+名称+类型+约束+描述”的格式，把能力转化为结构化模型：

#### 1. 属性定义
| 标识符          | 名称       | 数据类型 | 单位 | 读写权限 | 取值范围  | 描述                     |
|-----------------|------------|----------|------|----------|-----------|--------------------------|
| device_model    | 设备型号   | string   | -    | R        | -         | 温湿度传感器型号，如TH100 |
| firmware_version| 固件版本   | string   | -    | R        | -         | 设备固件版本，如V1.2.0   |
| temperature     | 实时温度   | float    | ℃    | R        | -40~85    | 设备采集的实时温度值     |
| humidity        | 实时湿度   | float    | %RH  | R        | 0~100     | 设备采集的实时湿度值     |
| sample_frequency| 采样频率   | int      | s    | RW       | 5~300     | 温湿度采样间隔，默认10s  |

#### 2. 服务定义
| 标识符          | 名称       | 入参                          | 出参                  | 调用方式 | 描述                     |
|-----------------|------------|-------------------------------|-----------------------|----------|--------------------------|
| reboot          | 重启设备   | -                             | result（bool）：是否成功 | 同步     | 触发设备重启，重启后5s上线 |
| calibrate       | 校准温湿度 | temp_offset（float）：温度偏移<br>hum_offset（float）：湿度偏移 | result（bool）、msg（string） | 同步     | 校准温湿度采集偏差       |
| set_sample_freq | 设置采样频率 | freq（int）：目标频率（5~300） | result（bool）、msg（string） | 同步     | 修改设备采样间隔         |

#### 3. 事件定义
| 标识符          | 名称       | 级别   | 携带数据                          | 触发条件                     |
|-----------------|------------|--------|-----------------------------------|------------------------------|
| temp_overlimit  | 温度超限   | 警告   | current_temp（float）、threshold（float） | 实时温度>设定上限 或 <设定下限 |
| hum_overlimit   | 湿度超限   | 警告   | current_hum（float）、threshold（float） | 实时湿度>设定上限 或 <设定下限 |
| low_battery     | 低电量     | 警告   | battery_level（int）：剩余电量（%） | 电量<20%                     |
| device_offline  | 设备离线   | 错误   | offline_time（string）：离线时间   | 设备30s未上报心跳            |

### 步骤3：补充元信息（版本/分类）
为模型添加全局元信息，方便管理和迭代：
- 模型ID：唯一标识（如`sensor.th温湿度.1.0`）；
- 模型版本：V1.0（后续新增能力可升级为V1.1）；
- 设备分类：环境传感器 > 温湿度传感器；
- 厂商信息：设备厂商、型号归属。

### 步骤4：验证与迭代
- 验证：对照设备实际能力，检查是否有遗漏（如是否支持校准）、错误（如温度取值范围是否合理）；
- 迭代：设备固件升级后（如新增“气压”采集），同步更新模型版本，新增`pressure`属性，不修改原有字段。

## 四、实战示例：JSON格式的物模型（通用IoT平台兼容）
多数IoT平台支持JSON格式导入物模型，以下是简化版示例：
```json
{
  "modelId": "sensor.th温湿度.1.0",
  "modelVersion": "1.0",
  "deviceType": "环境传感器",
  "properties": [
    {
      "identifier": "temperature",
      "name": "实时温度",
      "dataType": "float",
      "unit": "℃",
      "accessMode": "R",
      "range": [-40, 85],
      "description": "设备采集的实时温度值"
    },
    {
      "identifier": "sample_frequency",
      "name": "采样频率",
      "dataType": "int",
      "unit": "s",
      "accessMode": "RW",
      "range": [5, 300],
      "defaultValue": 10,
      "description": "温湿度采样间隔，默认10s"
    }
  ],
  "services": [
    {
      "identifier": "set_sample_freq",
      "name": "设置采样频率",
      "inputParams": [
        {
          "identifier": "freq",
          "name": "目标频率",
          "dataType": "int",
          "range": [5, 300]
        }
      ],
      "outputParams": [
        {
          "identifier": "result",
          "name": "操作结果",
          "dataType": "bool"
        },
        {
          "identifier": "msg",
          "name": "结果描述",
          "dataType": "string"
        }
      ],
      "callType": "sync",
      "description": "修改设备采样间隔"
    }
  ],
  "events": [
    {
      "identifier": "temp_overlimit",
      "name": "温度超限",
      "level": "WARN",
      "outputParams": [
        {
          "identifier": "current_temp",
          "name": "当前温度",
          "dataType": "float"
        },
        {
          "identifier": "threshold",
          "name": "阈值",
          "dataType": "float"
        }
      ],
      "description": "实时温度超出设定阈值时触发"
    }
  ]
}
